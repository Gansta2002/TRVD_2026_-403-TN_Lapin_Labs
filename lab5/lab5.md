## 2. HTTP-клієнт та Interceptor

Створено базовий Axios-клієнт у `services/api.js`.

Interceptor автоматично додає заголовок `Authorization: Bearer <token>` до кожного запиту якщо користувач залогінений.

При отриманні помилки `401 Unauthorized` — токен видаляється і користувач перенаправляється на сторінку входу.

## 3. Автентифікація (UI)

### Реєстрація
- Поля: email, пароль (мінімум 8 символів)
- Валідація на клієнті перед відправкою
- Запит: `POST /api/auth/register`

### Вхід (Login)
- Поля: email, пароль
- Після успішного входу JWT токен зберігається у `localStorage`
- Запит: `POST /api/auth/login`

### Protected Routes
- Компонент `ProtectedRoute` перевіряє наявність токена
- Якщо користувач не залогінений — перенаправляє на `/login`
- Сторінки `/admin` та `/books` — захищені

## 4. Основний функціонал (CRUD UI)

### Список книг (Read)
- Відображення у вигляді карток (Grid)
- Завантаження з API при монтуванні компонента
- Loading spinner під час завантаження

### Створення / Редагування (Create / Update)
- Форма з полями: назва, автор, рік
- При редагуванні поля заповнені поточними даними
- Запити: `POST /api/books`, `PUT /api/books/:id`

### Видалення (Delete)
- Кнопка "Видалити" з діалогом підтвердження
- Запит: `DELETE /api/books/:id`

### Зворотний зв'язок інтерфейсу
- Loading spinner при завантаженні даних
- Toast-повідомлення при успіху або помилці

## 7. Відповіді на контрольні питання

### 1. Що таке SPA і чим відрізняється від MPA?

**SPA (Single Page Application)** — це вебдодаток, який завантажує одну HTML-сторінку і динамічно оновлює її вміст без перезавантаження.

**MPA (Multi Page Application)** — кожен перехід між сторінками викликає новий запит до сервера і повне перезавантаження сторінки.

| | SPA | MPA |
|---|---|---|
| Швидкість після завантаження | Висока | Низька |
| Перше завантаження | Повільніше | Швидше |
| SEO | Складніше | Простіше |
| Приклади | React, Vue, Angular | Звичайні PHP-сайти |

### 2. Що таке Virtual DOM?

**Virtual DOM** — це легка копія реального DOM у пам'яті JavaScript.

Коли стан компонента змінюється:
1. React створює новий Virtual DOM
2. Порівнює його зі старим (diffing)
3. Оновлює тільки ті елементи реального DOM, які змінились (reconciliation)

Це значно швидше, ніж маніпулювати реальним DOM напряму.

### 3. Як працює Routing на клієнті?

Клієнтський роутинг використовує **History API** браузера (`pushState`, `replaceState`) для зміни URL без перезавантаження сторінки.

Звичайне посилання `<a href="/books">` викликає повний HTTP-запит до сервера.
React Router перехоплює кліки і замість цього оновлює URL через History API та рендерить відповідний компонент.

### 4. Де безпечніше зберігати JWT токени?

**Найбезпечніший варіант — HttpOnly Cookies.**

| Варіант | XSS-атака | CSRF-атака | Зручність |
|---|---|---|---|
| localStorage | ❌ Вразливий | ✅ Захищений | Простий |
| sessionStorage | ❌ Вразливий | ✅ Захищений | Простий |
| HttpOnly Cookie | ✅ Захищений | ❌ Потребує CSRF-токен | Складніший |

У даній роботі використовується `localStorage` як навчальний варіант, але у продакшені рекомендовано HttpOnly Cookies.

### 5. Що таке CORS?

**CORS (Cross-Origin Resource Sharing)** — механізм безпеки браузера, який блокує HTTP-запити до іншого домену (origin).

Origin = протокол + домен + порт. Наприклад:
- `http://localhost:3000` (фронтенд)
- `http://localhost:5000` (бекенд) — **різні origin!**

Браузер блокує такі запити за замовчуванням. Щоб дозволити їх, сервер повинен повернути заголовок:
```
Access-Control-Allow-Origin: http://localhost:3000
```

### 6. State Management — коли що використовувати?

**Локальний стан (`useState`)** — достатньо коли:
- Стан потрібен тільки одному компоненту
- Форми, тогл відкрито/закрито, локальні дані

**Глобальний стан (Context / Redux)** — потрібен коли:
- Дані потрібні у багатьох компонентах (наприклад, інформація про користувача)
- Стан потрібно передавати через кілька рівнів компонентів (prop drilling)

У даній роботі використовується `AuthContext` для зберігання інформації про залогіненого користувача.
